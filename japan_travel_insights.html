Japan Travel Insights

Understanding Why (and How) People Want to Visit Japan.

From street interviews in Yamagata to a scalable data product for tourism strategy.



Context & Business Problem

During a stay in Japan, I collaborated with 360, a company working with the Yamagata prefecture to understand and develop international tourism.

Initially, the team was running:

  • Long, face-to-face interviews with tourists (15‚Äì20 minutes each).
  • On-site data collection in touristic spots.
  • Highly manual and time-consuming operations, with no guarantee of volume (some days, very few or no foreign tourists).

Business issues:

  • High effort, low scalability.
  • Data limited to people physically present on-site.
  • Difficult to get a broader view of potential visitors who have not yet visited Japan or Yamagata.

My Solution

I designed and implemented an end-to-end data pipeline and analytics application to:

  • Move from local, manual interviews to online, scalable data collection.
  • Provide a living dataset that can grow over time.
  • Give the 360 team a self-service dashboard to explore insights and export data.

The solution combines:

  • Google Forms & Google Sheets for survey distribution and centralization.
  • Python for data cleaning and standardization.
  • GitHub Actions for scheduled data processing.
  • Streamlit for a fully interactive web dashboard.

Data Collection Strategy

Dual-Language Online Survey

I recreated and optimized the original questionnaire as two online forms:

  • One in French.
  • One in English.

Key improvements:

  • Smarter conditional logic (e.g. no need to ask detailed questions about Japan if the respondent has zero interest in visiting).
  • Better segmentation questions (motivations, perceived barriers, destination preferences, budget, etc.).
  • Focus on understanding why people choose Japan or alternative destinations.

I then shared the survey with:

  • My personal network and students.
  • My LinkedIn community.
  • Content creators and YouTubers with an audience interested in Japan.

From Raw Responses to a Clean Dataset

.1 Centralization

  • Each form (FR/EN) stores responses in its own Google Sheet.
  • A third sheet uses IMPORTRANGE to automatically combine all responses into a single table.

.2 Data Cleaning & Standardization

I built a dedicated Python pipeline to:

  • Standardize languages and values (e.g. country names, age groups, motivations).
  • Harmonize answer formats from both questionnaires.
  • Map free-text responses into broader analytical categories when relevant.

The result is a single cleaned dataset (df_clean.csv), ready for analysis and reporting.

Automated Refresh: A Living Dataset

To avoid manually updating the data whenever new responses arrive, I implemented:

  • A scheduled GitHub Actions workflow that runs twice a month.

On each run, the workflow:

  • Fetches the latest responses from the combined Google Sheet.
  • Applies all cleaning and transformation rules.
  • Overwrites the processed CSV file in the repository.

Why bi-monthly?

  • The survey is not time-critical.
  • A 15-day refresh cycle is more than enough to keep the dashboard reasonably up to date.
  • I can still trigger the workflow manually when needed.

Interactive Dashboard for 360

I built a Streamlit web application to make the analysis accessible to non-technical stakeholders.

Main features:

  • Interactive filters (e.g. nationality, age group, interest level, preferred destinations, fears or barriers).
  • Visualizations showing:
    • Who is interested in visiting Japan.
    • Which regions or cities are most attractive.
    • Why some respondents prefer other destinations.
    • What blocks potential visitors (cost, distance, language, safety, lack of information, etc.).
  • Download options:
    • Full ‚Äúraw‚Äù dataset.
    • Filtered subsets based on current selections (so 360 can run their own analyses in Excel, Power BI, etc.).

Performance-wise:

  • Because the data only changes twice a month, the app uses caching with a 15-day time-to-live.
  • This keeps the dashboard fast while still reflecting new data automatically when available.
